# ডকার কন্টেইনার  

ডকার (Docker) হলো আধুনিক সফটওয়্যার ডেভেলপমেন্টের একটি অপরিহার্য টুল। এটি অ্যাপ্লিকেশন ডেভেলপমেন্ট, শিপিং এবং রানিং প্রক্রিয়াকে অনেক সহজ করে তুলেছে।

---

## ১. ডকার কী? 
ডকার হলো একটি ওপেন-সোর্স প্ল্যাটফর্ম যা **Containerization** প্রযুক্তি ব্যবহার করে। এটি আপনার কোড, লাইব্রেরি এবং ডিপেন্ডেন্সিগুলোকে একটি প্যাকেজে রূপান্তর করে, যাকে বলা হয় **Docker Image**। এই ইমেজটি যখন রান করা হয়, তখন তাকে বলা হয় **Docker Container**।



---

## ২. ডকারের ইতিহাস 
* **সূচনা:** ডকার ২০১৩ সালে সলোমন হাইক্স (Solomon Hykes) এবং তার দল 'dotCloud' নামক প্রতিষ্ঠানে কাজ করার সময় তৈরি করেন।
* **ওপেন সোর্স:** এটি ২০১৩ সালের মার্চ মাসে সাধারণের জন্য উন্মুক্ত করা হয়।
* **বিবর্তন:** আগে লিনাক্স কন্টেইনার (LXC) ব্যবহার করা হতো যা বেশ জটিল ছিল। ডকার সেটিকে সহজ এবং পোর্টেবল করে তুলেছে।

---

## ৩. কেন ডকার প্রয়োজন? 
ডেভেলপারদের একটি কমন সমস্যা হলো **"It works on my machine"** (আমার পিসিতে চলে কিন্তু সার্ভারে চলে না)। ডকার এই সমস্যা সমাধান করে কারণ:
* **Consistency:** এটি নিশ্চিত করে যে কোডটি ডেভেলপমেন্ট, টেস্টিং এবং প্রোডাকশন—সব এনভায়রনমেন্টে একইভাবে চলবে।
* **Efficiency:** ভার্চুয়াল মেশিনের (VM) তুলনায় ডকার অনেক কম রিসোর্স (RAM/CPU) ব্যবহার করে।
* **Isolation:** একটি কন্টেইনারের ভেতর থাকা অ্যাপ অন্য কোনো অ্যাপের সাথে সংঘর্ষ (Conflict) তৈরি করে না।



---

## ৪. ডকারের ব্যবহার 
* **Microservices:** বড় প্রজেক্টকে ছোট ছোট অংশে ভাগ করে চালাতে।
* **CI/CD Pipeline:** অটোমেটেড সফটওয়্যার ডেলিভারি নিশ্চিত করতে।
* **Cloud Deployment:** খুব সহজে AWS, Azure বা Google Cloud-এ অ্যাপ ডিপ্লয় করতে।

---

## ৫. সুবিধা ও অসুবিধা 

| সুবিধা  | অসুবিধা |
| :--- | :--- |
| **Portability:** যেকোনো ওএস-এ চলে। | **Security:** কার্নেল শেয়ার করার কারণে কিছুটা ঝুঁকি থাকে। |
| **Speed:** খুব দ্রুত স্টার্ট হয়। | **GUI Apps:** গ্রাফিক্যাল অ্যাপ চালানো কঠিন। |
| **Scalability:** সহজে বাড়ানো বা কমানো যায়। | **Storage:** ইমেজ ফাইলগুলো অনেক সময় বড় জায়গা নেয়। |

---

## ৬. ডকার কীভাবে কাজ করে? 

ডকার ব্যবহারের ৪টি প্রধান ধাপ রয়েছে:
1. **Dockerfile:** অ্যাপ্লিকেশনের সব ইনস্ট্রাকশন যেখানে লেখা থাকে।
2. **Docker Image:** ডকার ফাইল থেকে তৈরি করা একটি স্ন্যাপশট।
3. **Docker Registry:** যেখানে ইমেজ জমা রাখা হয় (যেমন: Docker Hub)।
4. **Docker Container:** ইমেজের একটি জীবন্ত বা রানিং ভার্সন।

---

## ৭. গুরুত্বপূর্ণ কিছু কমান্ড 

```bash
# ইমেজ বিল্ড করা
docker build -t my-app-name .

# কন্টেইনার রান করা
docker run -d -p 8080:80 my-app-name

# সব রানিং কন্টেইনার দেখা
docker ps

# কন্টেইনার বন্ধ করা
docker stop <container_id>

```

## ডকার তিনটি প্রধান টেকনিক্যাল ফিচারের ওপর ভিত্তি করে কাজ করে:

* Namespaces: এটি কন্টেইনারকে আইসোলেশন (আলাদা করা) দেয় যাতে এক কন্টেইনার অন্যটির প্রসেস দেখতে না পারে।

* Control Groups (cgroups): এটি নির্ধারণ করে একটি কন্টেইনার কতটুকু র‍্যাম বা সিপিইউ ব্যবহার করতে পারবে।

* Union File Systems: এটি লেয়ারের মাধ্যমে ইমেজ তৈরি করে, ফলে একই ফাইল বারবার ডাউনলোড করতে হয় না।
